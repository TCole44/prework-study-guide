HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Used to create the structure of a webpage
  • made up of elements that work together to provide a framework for all web pages
  • elements are usually composed of an opening tag and a closing tag represented with angle brackets (< >)

    • Types of HTML Elements

    • !DOCTYPE html- Declaration that lets the browser know to expect HTML on the page
    • html lang-"eng"- opening tab marks beginning of the html document and lang-"eng" tells browser language will be in english
    • head- contains metadata, or data about our webpage that the browser uses to effectively serve our webpage to users, not typically seen by user but it contains important information about accessibility, search engines, and performance
    • title- defines title for a webpage and is required for Search Engine Optimization(SEO)
    • style- defines the CSS styles associated with a website
    • base- defines the base URL for a webpage
    • link- connects an external resource to the HTML document
    • meta- defines metadata such as the character set, description, keywords, author, and viewport
    • charset- sets our character encoding
    • http-equiv- sets an HTTP header for content
    • viewport- sets the default view for our webpage
    • script- embeds client-side scripts in the HTML document
    • body- defines the body of the document, which contains the content that is visible to the user
    • div- creates a division in the page
    • span- create an inline division
    • h#- represents level of heading, like pressing tab key, only one h1 element
    • p- paragraph or block of text
    • ul- unordered lists
    • ol- ordered lists
    • li- list items
    • img- contains info about images that are displayed on the webpage
    • src- defines location of the image
    • br- creates a line of empty space or a line break between two blocks of content
    • a- anchor- creates links to the same webpage or other webpages
    • href- points to the URL for the link
    • header- appears at the top of the page and usually contains a navigation menu and/or introductory content
    • main- ??????
    • footer- appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation

CSS

  • CSS- Cascading Style Sheets- used to add style to a webpage, defines how webpage content is presented to users
  • defines the colors, aesthetics, visual layout, and organization
  • 3 ways to style a webpage using CSS: incline CSS, internal CSS style sheet, and external CSS style sheet
  • incline CSS- used to style an element directly in the HTML file
  • internal CSS style sheet- makes it possible to embed entire stylesheet directly in the HTML file using a style element in the head element
  • external style sheet- allows developers to keep all their CSS rules in a seperate file which makes design changes easier
  • link element placed in HTML head element when using ecternal CSS style sheet
  • CSS rule- Selector defines elements or attributes to which declarations will apply, declarations consist of the CSS property that we want to apply and the value of that property
  • margin- indicates how much space we want around the outside of an element
  • padding- adds space around the content inside an element
  • text-align- one way to center a title or heading on a webpage
  • display: block;- display property is assigned with block value which assigns each image to start n a new line
  • link rel="stylesheet" href="./assets/style.css"- link creates the connection to the external file, rel specifies the relationship between the current document and the linked document resource, href specifies the location of the external resource

git

  • version control system that lets you manage and keep track of each version of your project
  • use command line to access
  • different than regular bash commands, always starts with git
  • great for tracking changes locally but github allows us to keep track of projects remotely and collaborate with other using repositories
  • repositories are used to store, files that contain code, files that contain other info about the code, ie. a README file, and version history for all files in the repo
  • README files- used to pass along useful information to anyone who visits the repository on GitHub
  • .md- an extension o indicate a Markdown file, similar to .doc, except we have to use a special syntax for formatting
  • code editor(VS Code) used to write, view, and edi our code
  • branch- copy or version of the code that can be worked on independently in isolation so development changes do not disrupt other team members who are also working on the code and testing their solutions
  • main branch is default branch, we should not be doing any coding in the main branch
  • when we creat a new branch to add something new to our project we call it a feature branch
  • In the feature branch, we make commits of all the changes we make to the code while working. We add a commit message that is descriptive of what changed in the code, then we push it to our feature branch in the github repo
  • PR- Pull Request- used to merge a comparison branch into te base branch
  • PR Open- you, other developers, and collaborators on your team can view those changes you made to the code and then comment on it, request changes, and even have a conversation about your changes
  • Once PR is approved, you can merge it into the main branch, any new branches will be created from the updated main branch

      • git commands

    • git clone - clones github repository onto our computer
    • git status - check what branch we are currently on
    • git checkout -b feature/????? - create a new branch, git precedes every Git statement, checkout moves the working branch to a new branch, -b creates a new branch
    • git add -A - add adds modifications in the current working branch to the staging area, -A indicates that we want to add all changes
    • git commit -m "add ?????" - command with a commit message that describes what the commit contains
    • git pull origin ?????- checks if our local branch is in sync with the base branch in Github, pull receives a branch's modifications into the local environment,origin indicates the source of the pull will be in the Github repo,????? indicates the branch
